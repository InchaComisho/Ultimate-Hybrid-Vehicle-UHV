# UHV適応制御・安全制御システム

## 概要

UHV適応制御・安全制御システムは、UHVの冷却、ミスト、走行風回収、駐車時ミスト遮熱、水質安全、歩行者安全、視界安全、補助エネルギー機能を統合的に管理するための概念的制御層である。

ハードウェアだけでは不十分である。UHVには、道路安全、視界、水質、バッテリー状態、歩行者近接、故障検知を優先する候補ロジックが必要である。この文書は認証済み車両安全ソフトウェアではなく、自律安全が解決済みであるとも主張しない。人間が監督する試作方向であり、検証が必要である。

## なぜ制御システムが必要か

ミスト出力、走行風回収、駐車時補助エネルギーは、状況に応じて制御されなければリスクを生む可能性がある。試作では、速度、湿度、雨、路面濡れ、視界、水質、歩行者近接、車両挙動、バッテリー状態へ適応する必要がある。

制御層は、安全条件が満たされない場合に機能を低減または停止するべきである。冷却性能はフェイルセーフ停止より優先されてはならない。

## センサー入力

必要な候補センサー入力は以下である。

* 車速
* 外気温
* 湿度
* 風速・風向
* 視界 / 霧 / 雨検知
* 路面濡れ
* 歩行者・自転車の近接
* 後続車距離
* 制動・旋回状態
* GPS / ジオフェンシング
* 水質
* タンク残量
* 補助バッテリー状態
* 主バッテリーインターフェース状態
* 故障検知

## 速度適応制御モード

代表的な概念モードは以下である。

```text
0 km/h:
  駐車時ミスト遮熱モードは、湿度、視界、水質、バッテリー状態、濡れ面が安全な場合にのみ作動候補となる。

1-10 km/h:
  低速待機または渋滞モード。ミスト出力は非常に限定し、歩行者安全を最優先する。

10-40 km/h:
  都市低速冷却候補モード。乾燥、高温、視界良好、安全条件下でのみミスト作動候補となる。

40-60 km/h:
  出力低減モード。検証されていない限り、車外ミストは低減または停止する。

60 km/h超:
  概念上の基準では、視界、漂流、安全リスクのため車外ミストは停止する。

高速道路モード:
  将来、認証済み閉鎖ダクト方式が検証されない限り、車外ミストは停止を維持する。
```

## ミスト出力制御

ミスト出力は標準動作ではなく、許可制の機能として扱う。候補出力は以下である。

* 停止
* 待機
* 極低出力
* 低出力
* 低減出力
* 検証済み低速条件下でのみ許可

温度や乾燥度は考慮するが、すべての熱要求より安全インターロックを優先する。

## 駐車時ミスト遮熱制御

0 km/hでは、雨水回収式・駐車時ミスト遮熱モードは安全な駐車条件下でのみ作動候補となる。制御器は、処理済み水の水質、タンク状態、補助バッテリー状態、湿度、視界、風による流れ、濡れ面検知、歩行者近接を確認する必要がある。

水質が不明な場合、駐車時ミスト遮熱は停止を維持するべきである。

## 補助エネルギー管理

補助エネルギーシステムは、待機センサー、水質監視、低電力換気、通信、駐車時ミスト遮熱の待機を支援する可能性がある。主駆動用バッテリーを充電できると仮定してはならない。

主バッテリーとの接続には、認証済みBMS連携、DC-DC制御、逆流防止、絶縁、故障検知、法的承認が必要である。

## 安全インターロック

以下の場合、システムはミスト出力を停止または低減するべきである。

* 湿度が高すぎる
* 雨を検知した
* 視界が悪い
* 路面が濡れている
* 歩行者または自転車が近すぎる
* 強い制動中である
* 急旋回中である
* 速度が許可モード範囲を超えている
* 水質が安全でない
* 凍結条件を検知した
* バッテリー状態が低すぎる
* センサー故障を検知した

## フェイルセーフ状態遷移

代表的な概念状態遷移は以下である。

```text
OFF
↓
STANDBY
↓
SAFE_CHECK
↓
PARKING_SHIELD / LOW_SPEED_URBAN / REDUCED_OUTPUT
↓
MONITOR
↓
FAULT_DETECTED → FAIL_SAFE_OFF
```

フェイルセーフ状態では、ミストと非必須負荷を停止し、必要な診断と操作者への警告を維持する。

## 手動停止・整備モード

人間が監督する試作では、手動停止、整備ロックアウト、故障リセット手順、状態表示が必要である。整備モードでは、タンク作業、フィルター交換、センサー校正、電気点検、清掃中の誤作動を防ぐ。

## 検証要件

検証項目は以下である。

* センサー精度試験
* 状態遷移試験
* 水質異常の故障注入
* 雨・視界異常試験
* 路面濡れ停止試験
* 歩行者近接停止試験
* 制動・旋回試験
* 低バッテリー時挙動
* 電磁・電気安全レビュー
* 操作者の使いやすさレビュー
* 法規・規制レビュー

## 限界

これは概念的制御層である。量産可能でも認証済みでもなく、完全な自律安全システムでもない。実装には、自動車安全工学、冗長性、検証、サイバーセキュリティ、保守性設計、法的承認が必要である。

## まとめ

UHV適応制御・安全制御システムは、UHVモジュールを安全優先で協調させる制御層である。冷却や補助機能は常時作動ではなく、条件付き、人間監督、フェイルセーフ制御の対象として扱うべきである。
